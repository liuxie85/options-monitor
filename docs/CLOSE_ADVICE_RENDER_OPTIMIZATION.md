# Close Optimizer 通知 Markdown 渲染优化方案

## 现状诊断

### 当前渲染输出示例

```
### [lx] 平仓建议
- NVDA Put 2026-06-19 150P · 强烈建议平仓换仓
- 已锁定: 85.0% | 剩余DTE=14 | 剩余收益年化=5.0%
- 价格: 开仓权利金=2.5 | 平仓 mid=0.4
- 估算: 平仓后锁定收益 $210 | 剩余权利金 $40
- 理由: switch
- 优化器: 持有年化=5.0% → 替代候选年化=12.0% | 尾部风险=0.045
---
```

### 问题清单

| 问题 | 影响 | 严重程度 |
|------|------|----------|
| 每行信息密度过高，`|` 分隔符在移动端难以扫描 | 可读性差 | 高 |
| 所有行均为 `- ` 开头，无视觉优先级区分 | 用户无法一眼识别"该做什么" | 高 |
| `---` 分隔线在 Markdown 中是水平线，占用屏幕空间 | 干扰阅读 | 中 |
| `理由: switch` 技术术语，对运营人员不友好 | 理解门槛 | 中 |
| 数字格式不统一（`%`/`$`/纯数字混排） | 视觉杂乱 | 中 |
| `###` 标题在 Feishu 中不够醒目 | 区段边界模糊 | 低 |

## 设计原则

1. **行动优先**：用户打开通知的第一眼应该知道"该对哪个持仓做什么"
2. **移动端优先**：每行不超过 35 个汉字等效宽度，适合手机竖屏阅读
3. **向后兼容**：不改变 CSV 输出、不改变 domain 评估逻辑、不改变 notify_levels 机制
4. **Markdown-friendly**：只使用基础 Markdown 语法（`#`、`- `、`**`），兼容 Feishu 纯文本渲染
5. **可配置**：通过 `close_advice.render_style` 开关控制，默认启用新样式

## 方案详情

### 方案：分层卡片式渲染（推荐）

#### 1. 整体结构

```
### 🔴 平仓建议 (2)

🔴 换仓 NVDA Put 150P @ 06-19
  已锁定 85% · 剩余 14 天
  收益 +$210 (余 $40)
  💡 持有 5% → 替代 12% | 风险 0.045

🟡 平仓 TSLA Put 200P @ 06-19
  已锁定 70% · 剩余 21 天
  收益 +$140 (余 $60)
  💡 持有 3% | 风险 0.060 | 无替代
```

#### 2. 单持仓信息层级

| 层级 | 内容 | 格式 | 目的 |
|------|------|------|------|
| L1 行动行 | `🔴 换仓 NVDA Put 150P @ 06-19` | emoji + 动词 + 标的 + 类型 + strike + 到期 | 一眼识别动作和目标 |
| L2 时间行 | `  已锁定 85% · 剩余 14 天` | 2 空格缩进 + 核心时间指标 | 确认时间紧迫感 |
| L3 资金行 | `  收益 +$210 (余 $40)` | 2 空格缩进 + 收益 + 剩余权利金 | 量化决策价值 |
| L4 优化器行 | `  💡 持有 5% → 替代 12% \| 风险 0.045` | 2 空格缩进 + 对比指标 | 支撑建议的数据依据 |

#### 3. Emoji 映射规则

| 场景 | Emoji | 说明 |
|------|-------|------|
| `optimizer_switch` | 🔴 | 最高优先级，需要立即行动 |
| `optimizer_close` | 🟡 | 高优先级，建议平仓 |
| `strong` (非 optimizer) | 🔴 | 现有 strong tier 保持红色 |
| `medium` (非 optimizer) | 🟡 | 现有 medium tier 保持黄色 |
| `optimizer_hold` | 🟢 | 建议继续持有，仅出现在详情 |
| 区段标题 | 根据最高优先级持仓决定 | 如果有一个 🔴，整个区段标题用 🔴 |

#### 4. 动词映射（替代技术理由）

| 原始 tier / optimizer_tier | 显示动词 | 示例 |
|----------------------------|----------|------|
| `optimizer_switch` | 换仓 | `🔴 换仓 NVDA Put...` |
| `optimizer_close` | 平仓 | `🟡 平仓 TSLA Put...` |
| `strong` | 强烈平仓 | `🔴 强烈平仓 NVDA Put...` |
| `medium` | 建议平仓 | `🟡 建议平仓 TSLA Put...` |
| `weak` | 考虑平仓 | `🟢 考虑平仓 ...` |
| `optional` | 可选平仓 | `⚪ 可选平仓 ...` |

#### 5. 日期格式化

- 当年内：`@ 06-19`（月-日，去掉年份）
- 跨年度：`@ 2026-06-19`（保留完整日期）

#### 6. 数字格式化

| 字段 | 格式 | 示例 |
|------|------|------|
| 百分比 | 整数（>=10%）或一位小数（<10%） | `85%`、`5.0%` |
| 货币 | `+$N` 或 `-$N`，去掉小数点后无意义的零 | `+$210`、`+$40` |
| 天数 | 纯数字 + "天" | `14 天` |
| 风险分数 | 3 位小数 | `0.045` |

### 新旧对比

**旧版（当前）**：
```
- NVDA Put 2026-06-19 150P · 强烈建议平仓换仓
- 已锁定: 85.0% | 剩余DTE=14 | 剩余收益年化=5.0%
- 价格: 开仓权利金=2.5 | 平仓 mid=0.4
- 估算: 平仓后锁定收益 $210 | 剩余权利金 $40
- 理由: switch
- 优化器: 持有年化=5.0% → 替代候选年化=12.0% | 尾部风险=0.045
---
```

**新版（提案）**：
```
🔴 换仓 NVDA Put 150P @ 06-19
  已锁定 85% · 剩余 14 天
  收益 +$210 (余 $40)
  💡 持有 5% → 替代 12% | 风险 0.045
```

**对比优势**：
- 行数从 7 行减少到 4 行（-43%）
- 首行直接给出"做什么"，旧版需要读到第 2 行末尾才知道行动建议
- 无 `---` 分隔线干扰
- 移动端每行长度从 ~60 字符降至 ~35 字符

## 实现计划

### 改动范围

只修改 `src/application/close_advice_runner.py` 的 `render_markdown` 和 `_optimizer_detail_lines`：
- 新增 `RENDER_STYLES` 常量（`legacy` / `compact`）
- 重构 `render_markdown` 支持两种样式
- 默认样式由 `close_advice.render_style` 配置控制，默认 `"compact"`
- 向后兼容：`legacy` 样式保持旧输出不变

### 不动的东西

- `domain/domain/close_advice.py`：评估逻辑、tier 判定、配置模型
- `src/application/close_advice_runner.py` 的 CSV 输出、`_selected_notify_rows`、输入处理
- `src/application/multi_tick/notify_format.py`：已有 emoji 高亮保持，不需要改动
- `tests/test_close_optimizer_notify.py`：通知格式高亮测试保持

### 测试策略

1. **契约测试**：新增 `test_render_markdown_compact_style` 验证新版输出格式
2. **回归测试**：旧版 `legacy` 样式测试保持不变
3. **端到端测试**：`test_close_advice_runner.py` 中验证 `close_advice.txt` 输出包含新版格式
4. **边界测试**：
   - 跨年度日期显示完整格式
   - 百分比 >=10% 显示整数
   - 无优化器时只显示 3 行
   - 多账户时每个账户区段独立

## 配置契约

```json
{
  "close_advice": {
    "enabled": true,
    "render_style": "compact",
    "notify_levels": ["strong", "medium"]
  }
}
```

- `render_style`: `"legacy"` | `"compact"`，默认 `"compact"`
- 不设置或设置为 `"legacy"` 时保持旧行为

## 风险评估

| 风险 | 概率 | 缓解措施 |
|------|------|----------|
| Feishu 对 emoji + 缩进排版支持不佳 | 低 | 缩进使用 2 个空格（纯文本安全），emoji 是 Unicode 标准字符 |
| 用户习惯旧格式 | 中 | 提供 `render_style: legacy` 回退 |
| 日期格式 `@ 06-19` 被误读 | 低 | 当年内统一用 MM-DD，跨年度用 YYYY-MM-DD，上下文有 option type 和 strike |
| 信息减少导致决策依据不足 | 低 | 开仓权利金和平仓 mid 从 L1-L4 移除，但这些信息在 CSV 中完整保留 |
